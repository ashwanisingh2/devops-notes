## Hinglish Writing Style
Explain in simple Hinglish. 
Theory sirf utni jitni practical samajhne ke liye required ho (80% Practical, 20% Theory). Avoid unnecessary history. Har difficult concept ko real-life example se explain karo.
Example: "DHCP automatically devices ko IP Address assign karta hai taaki manually IP dene ki zarurat na pade..."

## Note Structure
Every note must strictly follow this structure:

# Overview
Ye kya hai? Kyu use hota hai? Real life example. Industry kaha use karti hai? Real production use-case. Simple analogy. Architecture. Mermaid diagram.

# Working
Internal working. Data flow. Request flow. Authentication flow. Communication. Ports. Protocols. Dependencies.

# Installation
Prerequisites. Installation. Configuration. Verification. Rollback.

# Practical Lab
Step-by-step implementation. GUI Method. CLI Method. PowerShell. Bash. Azure Portal. AWS Console. Expected Output. Verification.

# Daily Engineer Tasks
L1 Engineer, L2 Engineer, L3 Engineer, Senior Engineer, Production Engineer, Cloud Engineer, DevOps Engineer, Microsoft 365 Engineer.

# Real Industry Tasks
Real tickets. Real change requests. Real maintenance work. Migration. Upgrade. Patch management. Health check. Production validation.

# Troubleshooting
Common issues. Symptoms. Possible root causes. Investigation steps. Commands. Logs. Resolution. Rollback. Verification. Escalation. Prevention.

# Interview Preparation
Basic, Intermediate, Advanced, Scenario Based, Production, HR Round, Manager Round, Rapid Fire, Hands-on Questions. Expected Answers. Confidence Level. Experience Level (L1/L2/L3).

# Production Scenarios
For scenarios like "Server down", "Website down", "Pod crash", explain: How to think. Where to check. Commands. Logs. Root Cause. Resolution. Verification.

# Commands
Command. Purpose. Syntax. Example. Output. When to use. When NOT to use. Danger Level.

# Cheat Sheet
Quick revision. Most important commands. Ports. Logs. Services. Registry. Configuration files. Interview shortcuts.

# SOP & Runbook & KB Article
SOP: Purpose. Scope. Procedure. Validation. Rollback.
Runbook: Detection. Investigation. Commands. Resolution. Validation.
KB Article: Problem. Environment. Symptoms. Cause. Resolution. Verification. References.

# Best Practices & Beginner Mistakes
Cover Microsoft, AWS, Azure, Google Cloud, Linux, Docker, Kubernetes, Security, Performance, Monitoring, Backup, HA, DR.
Beginner Mistakes: Common mistakes. Impact. Correct approach.

# Advanced Concepts
Internal architecture. Memory. Networking. Authentication. Authorization. Security. Performance optimization.

# Related Topics & Flashcards & Revision
Use Obsidian Wiki Links. Mention prerequisites and next learning topics.
Flashcards: Question/Answer.
Revision: 5 min, 15 min, 30 min, Interview revision.

# Real Production Logs & Commands & Decision Tree
Add sample logs and explain each line. Explain every command deeply. Create decision trees for troubleshooting.

# Visuals
Whenever useful create: Mermaid Diagram, Flowchart, Architecture Diagram, Sequence Diagram, Mind Map, Decision Tree.

# AI Enhancement
Never leave notes incomplete. Automatically add missing concepts, production knowledge, real examples, troubleshooting, etc., without waiting for instructions.

## Knowledge Vault Rules
- Never overwrite existing notes (unless asked to refactor). Always improve.
- Merge duplicate notes. Maintain folder structure.
- Maintain backlinks and use Obsidian Wiki Links.
- Add tags, aliases, and YAML metadata (frontmatter).
- Make everything searchable and keep formatting clean.

Final output must look like a premium internal engineering knowledge base.
